경력기술서
TCK는 BPO 사업을 하는 기업으로, 웹/앱 구축 및 서비스를 제공하는 팀에서 퍼블리싱 작업을 했습니다.

사내 서비스부터 타사 서비스까지 B2B 사업의 여러 방면으로 참여했으며, 적응형과 반응형의 웹 사이트, 어드민, 웹챗, 하이브리드 앱 등 다양한 환경에서 퍼블리싱 작업을 했습니다. 이러한 작업을 통하여 모바일 및 여러 환경에 유연하게 대응할 수 있는 경험을 쌓을 수 있었고, 규모가 큰 프로젝트에 참여하여 웹 및 앱 퍼블리싱을 효과적으로 수행하는 역량을 키웠습니다.

퍼블리싱 작업을 진행할 때 재사용성과 유지보수성에 중점을 두며 효율적인 UI 컴포넌트를 설계하기 위해 다양한 시도를 했습니다. 합니다. 이를 통해 효율적인 UI 개발로 개발 생산성 향상에 기여함과 동시에 사용자에게는 일관된 경험을 제공 함으로써 UX 개선의 효과까지 이끌어 낼 수 있었습니다.

상기 경험들을 토대로 디자인 시스템을 구축하여 향후 개발 일정에 대비했습니다. 또한, UI/UX 및 인터렉션 디자인에 대한 관심과 흥미를 바탕으로 기획자, 디자이너, 개발자와 협업하며 커뮤니케이션 능력도 키웠습니다.

    • gsap.js를 활용한 메인페이지 인트로 영역 구현 (톱니바퀴 회전 및 부드러운 텍스트 노출)
    • wow.js를 이용하여 스크롤에 반응하는 컨텐츠 영역 (순차적으로 나타나는 동적 레이아웃)
    • Before와 After 이미지 비교를 위한 슬라이드 기능 구현
    • 적절한 인터렉션 애니메이션을 사용하여 기업 이미지 강조 및 브랜딩 강화
    • 디자인 변경 및 컨텐츠 미수급 등의 이슈에도 재사용 가능한 컴포넌트로 효율적인 프로젝트 완료

    기업 이미지와 브랜딩을 인터렉션으로 표현되는 사이트를 구현하는 것이 핵심이였습니다.

    첫 번째 요소로 메인페이지의 인트로 영역에서는 gsap.js의 timeline 기능을 활용하여, 스크롤 동작에 따라 톱니바퀴가 동적으로 회전하고, 이후에 텍스트가 부드럽게 노출되는 애니메이션을 구현했습니다.

    컨텐츠 영역은 wow.js를 활용하여 사용자가 스크롤할 때, 컨텐츠가 순차적으로 나타나면서 동적인 느낌을 제공함과 동시에 많은 양의 정보가 아닌 최소한의 컨텐츠를 노출시켜 모든 내용을 다 볼 수 있도록 사용자 경험을 향상시켰습니다.

    두 번째로, 클리닝 서비스의 Before와 After 이미지를 명확하게 비교할 수 있는 기능을 구현했습니다. 이를 위해 input 요소의 range를 활용하여 반반 슬라이드를 개발하였습니다.

    마지막으로, 모바일 환경에서 서브 페이지의 사용자 경험을 개선하기 위해 드롭다운 탭을 구현했습니다. 화면 크기가 작아질 때, 탭이 자동으로 드롭다운 형태로 변환되어 사용자가 쉽게 이동할 수 있도록 했습니다.

    그 결과 메인페이지는 사용자에게 기업 이미지를 시각적으로 강조하여 전달하는 역할, Before&After 슬라이드는 기업 브랜딩을 강화 할 수 있는 효과를 줍니다. 기업의 온라인 존재감을 향상시키고 고객과의 상호작용을 개선하는 데 기여했습니다.

    또한 오픈 전 급작스런 디자인 변경 및 컨텐츠 미수급등 이슈가 있었으나 개발된 컴포넌트를 재사용하여 추가 작업을 최소한으로 유지하는 목표를 세운 후 원활한 커뮤니케이션과 효율적인 리소스 활용을 통해 프로젝트를 기한 내 성공적으로 완료했습니다.

    • 작업 툴 : Figma , Visual Studio Code
    • 컴포넌트 도입으로 재사용성 높임 (input, button 등)
    • 축소 가능한 사이드 메뉴와 유동적인 상단 탭 구현
    • chart.js를 활용한 메인 페이지 및 대시보드의 다양한 차트 구현
    • datatables.js를 사용하여 필터와 검색이 가능한 리스트 구현
    • TOAST UI Calendar.js를 이용하여 다양한 뷰 타입 지원하는 캘린더 구현
    • 모바일에서는 depth에 따라 다르게 스크롤되는 레이아웃 구현
    • 차트 라이브러리 및 컴포넌트 작업 경험 쌓음

    적응형 페이지로 제작되었으며, 매출 현황, 일정 관리, 고객사 관리 등을 편리하게 하기 위한 영업 관리 시스템입니다.

    이 프로젝트에서 처음으로 컴포넌트를 구현한 이유는, input과 button의 스타일이 사용처에 따라 다양하고 비슷한 레이아웃이 반복되며, 상태에 따라 스타일이 변하는 UI가 많았기 때문에 재사용성을 높이기 위함이었습니다.

    기본 레이아웃은 컨테이너 내에 많은 고정 레이아웃이 있어 컨텐츠 영역이 좁았습니다. 따라서, 사이드 메뉴를 축소할 수 있는 형태로 구성하고, 많은 페이지를 동시에 사용하는 경우를 고려하여 상단에 너비가 유동적인 탭을 구현했습니다. 이 탭의 디자인은 크롬 브라우저의 탭과 유사하며, 가상 클래스를 활용하여 자연스러운 곡선 형태를 유지했습니다.

    메인 페이지와 대시보드에서는 다양한 차트를 빠르고 쉽게 구현하기 위해 chart.js를 사용했습니다. 또한, 조회 영역의 리스트는 필터와 검색 기능이 필요하기에 초기 구현이 간단하면서도 다양한 기능이 있는datatables.js를 선택했습니다.

    캘린더 영역은 월별, 주별, 일별 등 다양한 뷰 타입을 지원해야 했고, 모바일까지 고려해야 했기에 TOAST UI Calendar.js를 사용했습니다. FullCalendar를 사용할 수도 있었지만, date picker UI도 함께 사용하기 위하여 tui를 채택하였습니다.

    모바일에서는 depth 별 스크롤 시 컨텐츠 영역을 다르게 주는 레이아웃을 구현했습니다. 1 depth에서는 헤더를 숨기고 검색, 필터 등의 툴바를 고정하여 컨텐츠 영역을 확장하며, 2 depth에서는 메뉴명을 감추고 타이틀을 배치하여 사용자가 더 편리하게 컨텐츠를 확인할 수 있도록 구성했습니다.

    이 프로젝트를 통해 차트 라이브러리 사용법을 숙지하고, 프로젝트 전반에 걸쳐 컴포넌트 작업을 지속해서 진행했습니다. 이후 여러 프로젝트에서의 컴포넌트 재사용성을 높이기 위해 SCSS를 학습하게 되었으며, 이는 더 유연한 컴포넌트 제작의 계기가 되었습니다.
