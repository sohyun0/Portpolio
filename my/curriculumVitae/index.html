경력기술서
TCK는 BPO 사업을 하는 회사로 비즈니스 IT 솔루션 및 웹/앱 구축과 유지보수 서비스를 제공하는 팀에서 퍼블리싱 작업을 했습니다.

자사 서비스부터 여러 엔터프라이즈의 서비스까지 여러방면으로 참여했습니다. 적응형과 반응형의 웹 사이트, 어드민, 웹챗, 하이브리드 앱 등 다양한 환경에서 퍼블리싱 작업을 했습니다.
이러한 작업들을 통하여 모바일 및 여러 환경에 유연하게 대응 할 수 있는 경험을 쌓을 수 있었습니다.

규모가 큰 프로젝트에 참여하여 효과적으로 웹 및 앱 퍼블리싱을 하는 역량을 키웠습니다.
HTML, CSS, SCSS, JavaScript와 같은 웹 기술을 사용했고, 작업할 때의 재사용성 및 유지보수를 고려하여 컴포넌트 작업을 많이 하였으며, 그 결과 동일한 ui를 사용함으로써 사용자 경험도 개선하고 개발 속도를 증진시키는 등 중요한 역할을 하였습니다.

컴포넌트 작성 경험을 기반으로 디자인 시스템을 구축하고, 향후 개발 일정에 대비했습니다.
또한, UI/UX에 대한 관심과 인터랙션 디자인에 대한 흥미를 바탕으로 기획자와 디자이너, 개발자와 협업하면서 커뮤니케이션 능력도 많이 키울 수 있었습니다.

  • [TCK] Design System 신규 구축 23.08 ~ 23.10

    • 유형 : 적응형 페이지 (데스크탑)
    • 기여도 : 컴포넌트 개발 및 퍼블리싱 100% , UI/UX 30%
    • 작업인원 : 퍼블리셔 1 디자이너 2
    • 구성 : html, css(scss), javascript
    • 작업 툴 : Figma , Visual Studio Code

    디자인 시스템을 구축하며 다양한 기업의 primary 및 secondary color를 고려하여 UI를 조정할 수 있는 유연한 시스템을 만들었습니다. 이로써 UI의 일관성을 확보함과 동시에 재사용성을 극대화하여 향후 프로젝트의 작업 공수를 줄이는 등의 효율을 향상시키는데 초점을 맞췄습니다.

    Foundation으로는 Typography, Color, Icon, Grid, Spacing이 포함되어 있습니다. 더불어 Input Fields, Drop Down, Radio, Checkbox, Button, Tooltip, Table, Tab, Calendar 등 다양한 컴포넌트도 포함되어 있어 다양한 프로젝트에서 사용할 수 있습니다. SCSS의 mixin과 변수를 적극 활용하여 색상, 버튼 크기, 폰트 스타일 등과 같은 기본 스타일을 다양한 색상 및 스타일에 대응할 수 있도록 설계했습니다.

    또한, 테이블 레이아웃부터 반응형 레이아웃까지 고려하여 12 Grid Layout을 Flex를 사용하여 만들었습니다. 프로젝트마다 변경되는 primary 및 secondary color를 적용하기 위해 아이콘은 선 모양 혹은 사각형, 원 형태와 결합한 형태로 디자인되었습니다. 이렇게 함으로써 CSS의 mask-image 속성을 활용하여 배경색만 변경하면 해당 primary color에 맞는 아이콘을 사용할 수 있게 만들었습니다.

    자주 사용되는 기능인 Calendar, Drop Down, Tab은 JavaScript로 간편하게 작동하며 커스터마이즈 할 수 있도록 개발했습니다. 특히, 캘린더는 기본 형태를 포함한 date picker 및 date range picker 기능을 포함한 형태로 구현하였는데 라이브러리를 사용하지 않고 직접 구현한 이유는 스타일 커스터마이즈가 어려운 경우에 대비하여 직접 개발하였습니다. 캘린더의 실제 내용을 표시하는 부분은 JavaScript를 이용하여 구현하였고, 내비게이션 부분은 자유롭게 커스터마이즈 할 수 있도록 설계했습니다.

    개별 컴포넌트를 활용할 수 있을 뿐만 아니라, 모듈 단위의 레이아웃을 구성하여 프로젝트 개발 속도를 높이기 위해 노력했습니다. 이로써 단순한 기능을 개발하는 데 많은 시간을 들이지 않고도 기존 모듈을 조합하여 다양한 레이아웃을 구현할 수 있습니다.

    또한, 두 가지 유형의 디자인 시스템을 구축하면서 UI/UX에 대한 이해도를 높이고 여러 스타일의 디자이너와 원활하게 협업할 수 있었습니다. 이 결과로 원하는 UI/UX를 만족시키는 결과를 얻었으며, 디자이너와의 협업을 강화할 수 있는 경험이 되었다. 이후 해당 시스템을 활용하여 개발할 때 가벼운 수정 요청에 신속하게 대응하고 프로젝트를 효율적으로 개발할 수 있었다는 긍정적인 피드백을 받았습니다.

  • 디비엘 코리아 홈페이지 신규 구축 22.10 ~ 22.12

    • 유형 : 반응형 페이지
    • 기여도 : 퍼블리싱 100% (1차 오픈)
    • 작업인원 : 퍼블리셔 2 디자이너 2
    • 구성 : html, css(scss), javascript , gsap.js (v3.11.3), wow.js (v1.1.2)
    • 작업 툴 : Figma , Visual Studio Code

    기업 이미지와 브랜딩을 강조하고 동적 디자인을 표현하는 데 중점을 두는것이 이번 프로젝트의 핵심입니다.

    첫 번째 요소로 메인페이지의 인트로 영역에서는 gsap.js의 timeline 기능을 활용하여, 스크롤 동작에 따라 톱니바퀴가 동적으로 회전하고, 이후에 텍스트가 부드럽게 노출되는 애니메이션을 구현했습니다.
    컨텐츠 영역은 wow.js를 활용하여 사용자가 스크롤할 때, 컨텐츠가 순차적으로 나타나면서 동적인 느낌을 제공함과 동시에 많은 양의 정보가 아닌 최소한의 컨텐츠를 노출시켜 모든 내용을 다 볼 수 있도록 사용자 경험을 향상시켰습니다. 이렇게 디자인된 메인페이지 영역은 사용자에게 기업 이미지를 시각적으로 강조하여 전달하는 역할을 합니다.

    두 번째로, 클리닝 서비스의 Before와 After 이미지를 명확하게 비교할 수 있는 기능을 구현했습니다. 이를 위해 input 요소의 range를 활용하여 반반 슬라이드를 개발하였습니다.
    Before 이미지 위에 After 이미지를 배치한 후, input으로 슬라이드의 가로 길이를 조절하여 두 이미지를 비교할 수 있도록 구현했습니다. 이렇게 구현된 슬라이드는 클리닝 서비스의 효과적인 시각적 차이를 볼 수 있으며 , 기업 브랜딩을 강화 할 수 있는 효과를 줍니다.

    마지막으로, 모바일 환경에서 서브 페이지의 사용자 경험을 개선하기 위해 드롭다운 탭을 구현했습니다. 화면 크기가 작아질 때, 탭이 자동으로 드롭다운 형태로 변환되어 사용자가 쉽게 이동할 수 있도록 했습니다.

    동적 디자인을 이용하여 기업 이미지를 강조하는 것이 해당 프로젝트의 목표였는데 위에서 설명한 다양한 기술과 기능을 활용하여 이를 달성하였습니다.
    그 결과 기업의 온라인 존재감을 향상시키고 고객과의 상호작용을 개선하는 데 기여했습니다.

    또한 오픈 전 급작스런 디자인 변경 및 컨텐츠 미수급으로 인하여 오픈을 2회차로 나눔과 동시에 작업자의 교체 이슈가 있었으나 개발된 컴포넌트를 재사용하여 추가 작업을 최소한으로 유지하는 목표를 세운 후 원활한 커뮤니케이션과 효율적인 리소스 활용을 통해 프로젝트를 기한 내 성공적으로 완료했습니다.

  • 전자랜드 영업 & 고객관리 플랫폼 신규 구축 22.05 ~ 22.10

    • 유형 : 적응형 페이지 (데스크탑 , 모바일), 하이브리드 앱 ( 태블릿)
    • 기여도 : 컴포넌트 및 모듈 제작 기여도 100% , Desktop 퍼블리싱 기여도 80% , Mobile 퍼블리싱 기여도 100%
    • 작업인원 : 퍼블리셔 2 디자이너 2
    • 구성 : html, css(scss), javascript ,chart.js (v2.9.4), realgrid.js (v2.5.4), jQuery UI (v1.12.1)
    • 작업 툴 : Figma , Visual Studio Code

    이번 프로젝트는 사내용 영업 및 고객 관리 시스템의 개발이 주요 목표였습니다. 이 시스템은 하이브리드 앱과 웹 사이트로 제작되었으며 고객 등록, 상담 내역 조회, 상품 가격 비교와 같은 고객 관리 기능과 판매 정산, 주간 및 월간 대시보드 및 리포트 조회와 같은 영업 관리 기능이 있습니다.

    우선, 퍼블리싱 기간이 규모에 비해 비교적 짧게 책정되었으며, 해당 일정을 고려하여 UI 설계를 모듈과 컴포넌트 단위로 구성하여 작업자들이 빠르게 작업을 진행할 수 있도록 했습니다. 이를 위해 해당프로젝트에서 처음으로 scss를 도입했습니다. 프로젝트가 크고 foundation 요소가 많아 실제 작업기간 전에 미리 mixin과 variable 설정을 해두어 이를 활용하여 기간을 짧게 사용 할 수 있었습니다. 또한 scss는 css와 문법적으로도 유사하기 때문에 빠르게 적용할 수 있었고, 상속이 편리해 가독성에도 높은 효율을 볼 수 있어서 도입을 결정했습니다.

    컴포넌트 단위의 UI에서 중점을 둔 부분은, 데이터 입력 부분이 종류가 많고 복잡했기 때문에 다양한 레이아웃과 디자인 요소를 고려하여 UI를 설계했습니다 input 필드에 자주 사용되는 prefix와 suffix를 쉽게 변경할 수 있도록 data-set을 활용하였고, input에 select, 아이콘이 결합되는 형태도 자주 사용되어 사용성이 떨어지지 않도록 wrapper의 최소영역을 320로 지정을 한 후 작업을 했습니다.
    버튼은 스타일 타입을 정해놓고 border-color , background-color는 class를 사용하여 쉽게 입힐 수 있도록 했습니다. 버튼 유형이 바뀌더라도 별도의 css 수정 없이 클래스 수정만으로 쉽게 바꿀 수 있습니다. 이와 같이, 데이터 입력의 다양성과 프로젝트 일정을 고려하여 효율적이고 편리한 UI를 설계하였습니다.

    모듈 단위의 UI는 아래와 같은 방법으로 구성하였습니다.

    첫 번째 홈화면에는 차트와 캘린더를 통해 통계 정보를 시각화하고 오늘의 목표 및 확인해야 할 사항을 표시하는 기능을 구현했습니다. 라인 그래프와 도넛 차트는 chart.js를 사용하여 생성하였고, 도넛 차트에는 커스텀 레이아웃을 적용하여 중앙에 텍스트를 표시했습니다. 라인 그래프는 가독성을 높이기 위해 데스크탑에서는 마우스 오버 시, 모바일에서는 클릭 시 툴팁을 통해 데이터를 확인할 수 있도록 구현하였습니다.

    캘린더에서 요일별 및 주간별 매출 실적을 확인하기 위해 디자인된 UI와 적합한 라이브러리가 없어 직접 캘린더를 개발했습니다. 많은 기능을 포함하고 있는 UI 개발은 새로운 경험이었으며 추가 UI 생성에서 몇 가지 어려움을 겪었습니다.
    추가 UI의 해결법은 캘린더 랜더링 시 당월 주차를 계산을 한 후 주간별 라인을 생성 하였고 요일별은 7일이 고정이기 때문에 캘린더와 별개로 하단에 따로 생성하는 방법을 사용하여 해결했습니다

    트리 구조의 메뉴 관리 기능은 텍스트 클릭 시와 체크박스 클릭 시 서로 다른 동작을 해야 했습니다. UI를 구현하기 위해 before에 미들 라인을 만들고 텍스트 클릭 시 하위 메뉴를 펼치는 버튼, 체크박스 클릭 시 체크만 되도록 하는 기능을 추가했습니다.

    두 번째로, 조회 영역에서는 그리드 기능을 필요로 했고, 기존 시스템에서 사용되는 리얼그리드 라이브러리를 적용하였습니다. 리얼그리드 스타일을 프로젝트의 디자인 시스템에 맞게 수정하여 통일성을 주었습니다

    세 번째로, 데이터 확인 영역은 대부분 표 형태로 구현했습니다.
    간단한 표 형태의 제목과 컨텐츠가 번갈아 나오는 구조는 라운드 테이블로, 제목이 상단에 위치하고 컨텐츠가 하단에 있는 구조는 기존 테이블 형태를 유지하는 컴포넌트를 제작하여 용도별로 유연하게 활용할 수 있도록 설계했습니다.
    상담 영역에서 컬럼이 동적으로 추가될 수 있는 경우, flex를 활용하여 테이블과 유사한 레이아웃을 제작했습니다.

    또한, 사용자가 테이블을 클릭하여 데이터를 수정해야 하는 요구사항이 있었으므로 contenteditable 속성을 활용하여 편집 가능한 텍스트를 적용했으며, 결제 부분에서 가장 중요한 부분은 최종 혜택금액과 최종 결제금액으로, 이 두 항목을 지속적으로 확인해야 했습니다. 이를 위해 sticky 속성을 사용하여 해당 항목이 항상 화면 상단에 고정되도록 디자인했습니다.

    모바일 UI 디자인에 특히 중점을 두었던 부분은 버튼에 액티브 효과를 부여하는 것이었습니다. 이를 위해 touchstart 및 touchend 이벤트를 활용하여 사용자의 상호작용에 반응하는 기능을 구현했습니다. 또한, 사용자 경험을 개선하기 위해 플로팅 버튼은 키보드가 활성화된 상황에서 시야 방해를 최소화하고 작업을 보다 편리하게 수행할 수 있도록 사라지도록 구현하였습니다.
