하이브리드 앱과 웹 사이트로 제작되었으며 고객 등록, 상담 내역 조회, 상품 가격 비교와 같은 고객 관리 기능과 판매 정산, 주간 및 월간 대시보드 및 리포트 조회와 같은 영업 관리 기능이 있습니다.

우선, 퍼블리싱 기간이 규모에 비해 비교적 짧게 책정되었으며, 해당 일정을 고려하여 UI 설계를 모듈과 컴포넌트 단위로 구성하여 작업자들이 빠르게 작업을 진행할 수 있도록 했습니다. 이를 위해 해당 프로젝트에서 처음으로 scss를 도입했습니다.

프로젝트가 크고 foundation 요소가 많아 실제 작업기간 전에 미리 mixin과 variable 설정을 해두어 이를 활용하여 기간을 짧게 사용할 수 있었습니다. 또한 scss는 css와 문법적으로도 유사하기 때문에 빠르게 적용할 수 있었고, 가독성에도 높은 효율을 볼 수 있어서 도입을 결정했습니다.

컴포넌트 단위의 UI에서 중점을 둔 부분은, 데이터 입력 부분이 종류가 많고 복잡했기 때문에 다양한 레이아웃과 디자인 요소를 고려하여 UI를 설계했습니다. input 필드에 자주 사용되는 prefix와 suffix를 쉽게 변경할 수 있도록 data-set을 활용하였고, input에 select, 아이콘이 결합하는 형태도 자주 사용되어 사용성이 떨어지지 않도록 wrapper의 최소영역을 320px로 지정을 한 후 작업을 했습니다.

버튼은 스타일 타입을 정해놓고 클래스 조합만으로 쉽게 적용할 수 있도록 했습니다. 버튼 유형이 바뀌더라도 별도의 css 수정 없이 클래스 수정만으로 쉽게 바꿀 수 있습니다. 이처럼, 데이터 입력의 다양성과 프로젝트 일정을 고려하여 효율적이고 편리한 UI를 설계하였습니다.

모듈 단위의 UI는 아래와 같은 방법으로 구성하였습니다.

첫 번째 홈 화면에는 차트와 캘린더를 통해 통계 정보를 시각화하고 오늘의 목표 및 확인해야 할 사항을 표시하는 기능을 구현했습니다. 라인 그래프와 도넛 차트는 chart.js를 사용하여 생성하였고, 도넛 차트에는 커스텀 레이아웃을 적용하여 중앙에 텍스트를 표시했습니다.

캘린더에서 요일별 및 주간별 매출 실적을 확인하기 위해 디자인된 UI와 적합한 라이브러리가 없어 직접 캘린더를 개발했습니다. 많은 기능을 포함하고 있는 UI 개발은 새로운 경험이었으며 추가 UI 생성에서 몇 가지 어려움을 겪었습니다.
추가 UI의 해결법은 캘린더 렌더링 시 당월 주차를, 계산을 한 후 주간별 라인을 생성하였고 요일별은 7일이 고정이기 때문에 캘린더와 별개로 하단에 따로 생성하는 방법을 사용하여 해결했습니다.

두 번째로, 조회 영역에서는 그리드 기능이 필요했고, 기존 시스템에서 사용되는 리얼 그리드 라이브러리를 적용하였습니다. 리얼 그리드 스타일을 프로젝트의 디자인 시스템에 맞게 수정하여 통일성을 주었습니다

세 번째로, 데이터 확인 영역은 대부분 표 형태로 구현했습니다. 간단한 표 형태의 제목과 컨텐츠가 번갈아 나오는 구조는 라운드 테이블로, 제목이 상단에 위치하고 컨텐츠가 하단에 있는 구조는 기존 테이블 형태를 유지하는 컴포넌트를 제작하여 용도별로 유연하게 활용할 수 있도록 설계했습니다. 상담 영역에서 컬럼이 동적으로 추가될 수 있는 경우, flex를 활용하여 테이블과 유사한 레이아웃을 제작했습니다.

모바일 UI 디자인에 특히 중점을 두었던 부분은 버튼에 액티브 효과를 부여하는 것이었습니다. 이를 위해 touchstart 및 touchend 이벤트를 활용하여 사용자의 상호작용에 반응하는 기능을 구현했습니다. 또한, 사용자 경험을 개선하기 위해 플로팅 버튼은 키보드가 활성화된 상황에서 시야 방해를 최소화하고 작업을 더욱 편리하게 수행할 수 있도록 사라지도록 구현하였습니다.

이 프로젝트를 통해 프로젝트 일정을 준수하면서도 높은 품질과 효과적인 UI 설계와 구현을 하였고, 모듈과 컴포넌트 단위로 작업을 구성하여 팀원들 간의 협업을 원활하게 만들었습니다.

또한, 사용자 중심의 디자인을 강조하여 모바일 UI에서의 터치 이벤트와 키보드 활성화 시의 시야 방해를 최소화하는 노력을 통해 사용자 경험을 개선할 수 있었습니다. 이러한 경험을 바탕으로 향후 프로젝트에서도 편리한 UX와 효율적인 UI 개발을 지속해서 추구하고자 합니다.
